# 天津市市政工程局综合管理系统 - 自动化测试框架

## 1. 概述

本文档描述了天津市市政工程局综合管理系统的自动化测试框架架构设计。
本框架旨在实现**高覆盖率**（目标85%+）的全面测试，包括单元测试、API测试、数据库测试、UI测试、集成测试等多个维度。

## 10. 运行方式

### 10.1 运行所有测试

```bash
pytest tests/ -v
```

### 10.2 运行指定模块

```bash
# API测试
pytest tests/api/ -v

# 单元测试
pytest tests/unit/ -v

# UI测试
pytest tests/ui/ -v
```

### 10.3 生成报告

```bash
# 生成Allure报告
pytest tests/ --alluredir=reports/allure
allure serve reports/allure

# 生成覆盖率报告
pytest tests/ --cov=tianjin_bureau_system --cov-report=html
```

### 10.4 并行执行

```bash
# 使用4个进程并行执行
pytest tests/ -n 4
```

## 12. 维护指南

### 12.1 新增测试

1. 确定测试类型（API/UI/集成等）
2. 在对应目录创建/编辑测试文件
3. 使用现有的fixtures
4. 遵循命名规范: `test_<功能>_<场景>`

### 12.2 测试数据清理

- 每个测试用例后自动清理
- 使用 `@pytest.fixture` 的 `yield` 机制
- 关键数据使用 `db_fixture` 事务回滚

### 12.3 调试技巧

```bash
# 只运行失败的测试
pytest --lf

# 详细输出
pytest -vv -s

# 在第一个失败处停止
pytest -x

# 查看测试覆盖分布
pytest --cov-report=term-missing
```
