# 天津市市政工程局综合管理系统 - 自动化测试框架

## 1. 概述

本文档描述了天津市市政工程局综合管理系统的自动化测试框架架构设计。
本框架旨在实现**高覆盖率**（目标85%+）的全面测试，包括单元测试、API测试、数据库测试、UI测试、集成测试等多个维度。

### 7.3 三级管理权限验证

```
局机关处室测试场景：
  - 资产管理处 → 查看全部资产、统计、审批
  - 办公室 → 公文审核下达、消息通知(局名义)
  - 劳动人事处 → 账号注册审批、密码审批
  - 科技处 → 科研项目审批、科研经费管理
  - 财务处 → 预算审批、预算分配

中层单位测试场景：
  - 提交公文给机关处室
  - 接收机关下达的公文
  - 同级单位间发送"同级函件"
  - 审批基层单位申报
  - 向上级提交请示

基层单位测试场景：
  - 只与直属中层单位交互
  - 不能直接与机关处室通信
  - 向上提交公文、科研申报、预算申报
```

## 12. 维护指南

### 12.1 新增测试

1. 确定测试类型（API/UI/集成等）
2. 在对应目录创建/编辑测试文件
3. 使用现有的fixtures
4. 遵循命名规范: `test_<功能>_<场景>`

### 12.2 测试数据清理

- 每个测试用例后自动清理
- 使用 `@pytest.fixture` 的 `yield` 机制
- 关键数据使用 `db_fixture` 事务回滚

### 12.3 调试技巧

```bash
# 只运行失败的测试
pytest --lf

# 详细输出
pytest -vv -s

# 在第一个失败处停止
pytest -x

# 查看测试覆盖分布
pytest --cov-report=term-missing
```
